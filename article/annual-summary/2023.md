往后的一个月里面，就是每天早上去打个卡，白天的时间也就自由了，偶尔投投简历，写写自己的项目。本来也没着急，也就打算随缘找找，随便投投简历，找工作应该也没有那么难，4 月 1 日算是正式离职，本以为顶多四月底，也能找的工作了，可没成想，这一找，就找了**五个月**的工作。

## 离职期间

离职的前一个月，每天投投简历，不过基本上也都沉入大海了，投出去的也没有回复，白天就在家里打英雄联盟，下午做个饭，正好老婆下班了回来吃饭。

一个月过后，也正式离职了，游戏也不想再打了，就打算着自己能做点什么，尝试自己每天不用上班对着电脑也可以赚到钱，当时正值 AI 发展最火热的时候，就每天早上早早起床，背着书包带着电脑，早上九点钟左右到[星巴克](https://www.starbucks.com.cn/)研究一些 AI 相关的东西，再学习一些新的内容。每天在星巴克除去中午出去吃个饭是时间，一直都在对着电脑。到了下午三四点钟，就去楼上超市买个菜，回家做饭。

## 投递情况

投递简历起初是使用 BOSS 直聘，后来也用了 脉脉、智联等软件，但是除了 BOSS 之外，其它的软件感觉真都不怎么样。BOSS 一天大概可以投递 180 多个简历，起初还是有选择性的投递，看看那些公司比较合适，但是这样就导致基本上没有人回复，很多时候一天投递到上限，回复的也就一两个，更别说面试了，在一个半月的时间里，我没有约到过面试。

所以后来也就随缘了，每天海投，直接从第一个开始投，一直投到上限，还注册了小号投递，一直到 8 月份找到工作，我一共投递出去了 **6000+** 简历。

其中，投递了好几种岗位，运营、销售、开发、快递员、货运司机、网约车、检票员、主播，都有面试过。去了各行各业面试，也让我感受到了和互联网相比，不一样的面试形式。比如我有面试过送货的骑手，面试的流程呢，就是上来让你填个表，然后等待面试，面试是一个面试官对两个人，进去坐下之后，面试官直接拿一根笔，在纸上边说边画，送 1 单多少钱，一天送多少单，然后如何打款，电动车如何租赁，介绍完之后，就问：明天能不能来上班？我当时说再考虑下，看我们两个都没啥问题，就说面试结束了，需要入职再点电话联系。

面试结束之后，我就很是震惊，居然这样传统行业面试居然这么草率，不像是开发面试卷的都那么厉害，这样的面试我还真有点不太习惯。

主播的话，虽然我之前做过，也可以做，但是玩的游戏内容真的是无聊透顶，很多游戏的玩法，我之前也没加过。竞技类型的游戏也不适合我了。

销售的话，我投递了特斯拉的销售，但是好像都需要经验才行，hr 只是给打过一次电话，就再也没有音讯了。

## 短视频

这个短视频的时代，有很多人通过短视频翻身，而且没有入门的门槛，所以我也打算开始尝试第一个副业：做短视频。其实我期望的副业就是可以 0 成本入手尝试，不要花一分钱，这样我就可以无限的试错。就算我没有成功，我也不会有任何损失，给我带来的只有收获，哪怕是失败。

其实我的想法是，我只要每天对着电脑，不需要有摄影师，不需要表演，就可以做出来视频，当时也是通过社交网络看到一个和 AI 融合，就可以做出短视频的方法，自己也开始研究。

当时就使用 ChatGPT 来写一些主题的文案，然后使用一个剪辑工具可以将文案生成视频，再使用一些绘画的 AI 工具画出视频主题的封面图，所有的声音也通过 AI 来合成。那时候的 AI 工具真的是五花八门，有收费的也有免费的，质量也是参差不齐，为了节约成本，我只能是找到一些质量较高的免费工具。

起初是使用剪映，将文案喂给剪辑软件，然后就会生成一个有图有声音有字幕的视频。第一个我做的是讲故事主题的号，并且发布到抖音。以我对抖音的理解：只要是一个新号，当发布了 10 个作品之后，播放量依然惨不忍睹的情况下，这个号也就没必要做下去了。然而我的号就被我做成了这样。

第一个主题失败了，后面我又打算做一个旅行相关的，找到一个城市或者景点，让 AI 写个介绍，然后生成视频，并且我尝试在视频中加入一些彩蛋，抖音和快手两个平台同时发布视频。而且还打算做一个旅行主题相关的小程序和官网，通过短视频平台的流量，推广小程序，在小程序中加入付费相关的内容，来实现收入。为了实现这一想法，还特意买了个新的域名，域名就是我这个项目的名字。AI 的视频生成工具我也找到了个质量更高一些的[度加](https://aigc.baidu.com/make)，做了一段时间之后，大概发布了 20 多个视频之后，也是惨不忍睹，也是不再更新了。

短视频这条路，运气成分太高了，看似入门门槛很低，但是早就饱和的都要溢出来了。

## 做个自己的产品

看来短视频这条路不太容易走通，并且运气成分太大。还是打算把自己开发者的知识运用一下吧。之前打算做个旅行相关的网站。正好之前有接触过一个 node 的后端框架 [nestjs](https://github.com/nestjs/nest)，并且 Mysql 也会一点，就开始配合 AI 来做个自己的产品，不会的东西使用 AI 来帮助实现。前端的话，就使用 vue3，组件库就用我自己的组件库 [fighting-design](https://github.com/FightingDesign/fighting-design)。在开发的途中，也会发现一些组件库的 bug，正好也可以修复掉其中的一些问题。两个项目一起完善。

我希望自己的网站是足够有特点的，也就是用户能看到的东西、交互上要实现和市面上的产品不一样，或许每个打算做自己产品的人都是这样想的，但是如何做？做成什么样？这是最让人头疼的，所以我就会经常看各式各样的网站，发现都不是我想要的效果，在每天睡前，或者坐地铁或坐车的时候，就一直在思考，什么样的交互才最有特色的。

其实有一些特殊的想法，不一定是要在相似的产品中找到答案。比如我想要做一个网站，不一定是在众多网站中找到灵感。也可以通过其它途径。比如我比较喜欢的 [北京冬奥会开幕式倒计时短片](https://www.bilibili.com/video/BV1Vr4y1z7Up/?spm_id_from=333.999.0.0&vd_source=45b9ebc52408dcecd5ef3a71d406552e)，冬奥会的开场倒计时，我感觉就是一个打破常规的操作：倒计时间隔不一定是 1 秒钟。使用二十四个节气，作为倒计时，依次播放，正好冬奥会是在立春，所以倒计时最后定格在了立春。除此之外，我参考的还有：杭州亚运会的开幕式、一些海报、特斯拉等。

最终，我借助了特斯拉的设计，想到了一个全新的创意，因为特斯拉的各种操作设计都是比较颠覆常规的，可以说是有些反人类，因为我想要做出来的效果就是：反人类设计，大脑洞的设计。所以，参考一些和你当前做的看似无关的东西，只要深入思考一下，想到一些类似的设计手法，获取可以得到不同寻常的结果，这个结果要么很糟糕，要么很亮眼，起码我不会让人说：“还行” 这种中等不上不下的词。我希望的是：要么最好，要么最差。我的作品在后来面试的时候，也惊艳到了面试官，为我拿到了 offer。

目前我的产品，还没有开源，所以我具体做了那些反人类的设计，目前也没有展示给更多的人。

后来我又开发了微信小程序版本，将 web 端的设计，换了一种可以在手机上交互的全新的形式来展现的。目前还在开发中，还未上线，近期也在一直完善中，希望 2024 年可以面向大家展示一下，哈哈。

### 后端和操作系统

4 月份的时候，自己学了个后端框架 [nestjs](https://github.com/nestjs/nest)，是一个基于 Node 的 MVC 框架，支持 TypeScript。现在觉得，选择学习 nest 真的是个非常正确的决定！

为什么呢？首先，身为前端工程师对 js 零成本上手，可以自己来实现一个全栈的项目，当然这还不是最主要的。更重要的是，在国外的生态中，nest 还是非常受欢迎的，不像是在国内，基本上任何一个后端都要选择 Java，首先体积太过于庞大了，而且人力成本很高。国外 node 相关的框架还是很受欢迎的，对于前端工程师一个人也可以实现一个完整的项目，有机会可以接到一些海外的兼职，非常不错。

后端有个必须要打交道的就是数据库，开始我使用的是 [Navicat](https://www.navicat.com/en/) 和 [宝塔](https://www.bt.cn/new/index.html)，图形化的数据库，后来感觉，老是使用图形化的界面，收获还是太小了，还是得自己学学 SQL，就放弃了图形化工具的使用。开始学习自己手写 SQL 和命令，不会的就使用 ChatGPT 学习一下，再把学到的 SQL 和命令记录成 [文档](https://blog.tianyuhao.cn/docs/server/server-2.html)。

另外的话就是操作系统，现在每天我使用三个操作系统，公司的电脑是 MacOS，我的笔记本是 Windows，我的服务器是 Linux，也正好将三个主流的操作系统全部了解一下。

### React

作为前端最主流的框架之一，React 之前一直没有接触，最近我也开始学习，为什么呢？因为 React 可以帮我赚到钱。只能说，现在什么东西可以帮我赚到钱，我就学什么，海外的很多岗位都是需要 React，后端大多数就是前面说的 nestjs，所以会了这两个技能，对于找到一些海外的工作机会机更多一些。也是最近才开始学的，并且把学习的内容也记录成为了[文档](https://blog.tianyuhao.cn/docs/react/react-1.html)。

React 相比于 vue 的区别呢。vue 是会给你定制很多规范，你必须按照它的规则来写，所以就会导致大家写的 vue 代码基本上都一样，大差不差。react 的话，就是完全给你自由，各个你要用的东西都需要你自己去配置，它内部的方法和 API 也非常少，配置完全开放。所以新手更适合 vue，但是会了 vue 之后，上手 react 很容易，一些方法看看就知道咋用了。

React 基础学完之后，目前在开始学学它的周边生态，比如 [react-router](https://reactrouter.com/en/main) 和 [zustand](https://github.com/pmndrs/zustand)，其它的就是一些 css 相关的处理方法。学起来还是比较愉快的。

## 作为面试官

11 月份，在新的公司工作的时候，还需要再招聘一名新的前端同学，我负责了在平台上发布职位和招聘面试的工作。一天可以收到好几十封简历。在不到一个月内，共九场面试，面试了 8 名新同学。大多数都是三到四年的工作经验。

面试我主要是会问一些技术相关的问题，另一个和我一起的同事会问一些项目问题。如果我们技术面通过了，会到产品面。

先说在平台上收到简历的质量，真的是啥样的都有，奇葩的也有，质量高的是真少。关于简历怎么写，我建议可以简单的参考下[大圣](https://github.com/shengxinjing)的视频 [程序员简历点评第二期：能吃苦和能吃](https://www.bilibili.com/video/BV1av411w7oa/?spm_id_from=333.788&vd_source=45b9ebc52408dcecd5ef3a71d406552e)，看完一定会有一定收获。就算没学到啥，也希望你工作三四年的简历上技术栈第一条也不要写会 HTML 和 CSS 了。

正常来说，一个面试官看一份简历，大概 20 秒钟，如果在这 20 秒钟没有让我对你产生情趣，那么就直接被筛掉了。如果在这几秒钟的时间里，我对你产生了兴趣，那么会继续仔细的看一看，发掘一下你的优点，如果还不错，那么就会提交上去作为下一轮筛选的候选人。

关于简历上写什么，先，放了 Github 地址，对于我来说绝对是加分项，我一定会点击进去看的，但是如果 1 年提交 10 几次，甚至没有。那么我就会直接筛掉，这就是放了地址还不如不放。本来我看了放 Github，加 10 分，一看根本没咋提交过代码，那直接-90 分。再一个就是有博客也要放上去，这绝对是加分项。还有的同学会把自己做的一些项目放上 git 地址，放 git 是很不错，但是 git 只有 1~2 次提交，这很明显就是为了放链接而提交，为了展示而提交，没有任何意义。

面试中，我作为纯技术面，每个人问的第一个问题就是：JS 的数据类型有几种？分别是什么？这样一个基础的问题，目前为止我只遇到一个人完全的答对，具体有那些，可以参考[JavaScript 数据类型和数据结构](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Data_structures)，之后我会问问比如：高阶函数是什么、高阶函数解决了什么问题、类如何继承、原型方法和静态方法、事件循环机制等。我比较看重的就是数据类型了解不了解，函数和类玩的 6 不 6。除了硬性技术之外的，我比较关注是否做过开源，对 AI 是否了解，比如：你觉得自己还要什么需要提升的、AI 相关的产品了解什么、有没有参与过开源项目等。

面试几个人下来，给我的感觉呢，其实我更希望面试者会更主动一些，比如主动自我介绍，主动介绍自己的产品，延伸或者展开一些话题，对各个事物有些自己的看法等。但是大多数求职者都没有做到这样，给我的感觉就是我在问，你在答，而不是感觉是在聊天。
